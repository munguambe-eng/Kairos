Type 02 · Structured Reports · Smurfing
Cumulative
Threshold
Relatórios por Acumulação Periódica

Multiple smaller transactions that individually fall below the threshold but cumulatively reach it over a defined period. Classic structuring — intentional splitting to evade detection. Kairos Lex tracks cumulative totals across daily, weekly, and monthly windows simultaneously.

Cascade Rule — No Double Reporting
Daily
→
Transactions that reached the threshold within the day are reported. They are flagged and excluded from all subsequent accumulation windows.
Weekly
→
Only transactions not yet reported in the daily window accumulate toward the weekly total. Already-reported transactions are excluded.
Monthly
→
Only transactions not yet reported in either daily or weekly windows accumulate toward the monthly total. Zero overlap guaranteed.
Separate accumulation counters per window — daily, weekly, monthly
Each transaction carries a report status flag — prevents double reporting
Pattern analysis flags clients with repeated sub-threshold behaviour
STR triggered automatically if structuring pattern is confirmed

STR Detection

Seven rules.
One score.

Every transaction is evaluated against all seven rules simultaneously. The composite score drives urgency and prioritisation. No minimum value — any suspicious transaction of any size qualifies for STR.

Rule 01
Structuring (Smurfing)
Amounts just below CTR / IFT / EFT thresholds — cumulative patterns detected across reporting periods
+40 pts
Rule 02
High-Risk Country (FATF)
Transaction involving FATF blacklist or grey-list jurisdiction
+35 pts
Rule 03
Enhanced Monitoring Country
Countries with AML/CFT deficiencies under enhanced FATF monitoring
+25 pts
Rule 04
Large Cash Without Justification
Cash deposit or withdrawal above threshold with no supporting narrative in the system
+30 pts
Rule 05
Round Number Amounts
Large transactions with perfectly round figures — a recognised indicator of pre-determined transfers
+20 pts
Rule 06
Suspicious International Transfer
SWIFT transactions to flagged corridors above minimum threshold
+30 pts
Rule 07
Statistical Anomaly (z > 3)
Amount more than 3 standard deviations above the client's own historical average
+25 pts
